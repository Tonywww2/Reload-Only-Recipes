Reload Only Data（选择性数据重载）是一款实用工具类模组，用于按内容类型单独重载数据包（datapack）内容，可分别重载配方、进度、标签、战利品表、函数，以及数据包注册表（实验性）。相较于执行完整的 /reload 指令重跑全部内容，本模组仅重载指定的一类内容，在大型整合包中可显著降低重载开销。其中配方重载额外提供对 KubeJS 运行时脚本修改的兼容。

本模组提供两个平台版本：Minecraft 1.20.1（Forge）与 Minecraft 1.21.1（NeoForge）。项目最初仅用于重载配方，后扩展为通用的选择性重载框架，并保留 /reloadrecipes 指令作为重载配方的快捷方式。

## 支持版本

| 加载器 | Minecraft 版本 | 加载器版本 | Java | KubeJS（可选前置） |
| --- | --- | --- | --- | --- |
| Forge | 1.20.1 | 47.4.4 及以上 | 17 | 6 代 |
| NeoForge | 1.21.1 | 21.1.234 及以上 | 21 | 7 代 |

KubeJS 为可选前置。未安装 KubeJS 时，配方重载采用原版策略；安装后按加载器平台自动启用对应的 KubeJS 6 代或 7 代兼容层。开发与编译期所用的 KubeJS 版本为 Forge 平台 2001.6.5-build.26 与 NeoForge 平台 2101.7.2-build.369，实际运行时以玩家安装的 KubeJS 版本为准。

## 指令

所有指令均需要 OP 权限（权限等级 2）。统一入口为 /reloadonly <目标> [参数]，并保留 /reloadrecipes 作为重载配方的快捷指令。

| 指令 | 重载内容 | 客户端同步 |
| --- | --- | --- |
| /reloadrecipes | 配方（兼容 KubeJS） | 是（配方与配方书） |
| /reloadonly recipes | 配方（兼容 KubeJS） | 是 |
| /reloadonly advancements | 进度 | 是（重建进度树并为在线玩家重新计算进度） |
| /reloadonly tags <注册表> | 指定注册表的标签 | 是 |
| /reloadonly loot | 战利品表、谓词与物品修饰器 | 否（纯服务端） |
| /reloadonly functions | 函数（.mcfunction） | 否（纯服务端） |
| /reloadonly registry <注册表> [confirm] | 单个数据包注册表（实验性） | 需重新连接客户端 |

指令的目标与参数均支持 Tab 补全。对未知或不支持的目标，指令会明确返回失败提示，而不会返回虚假的成功结果。指令反馈支持简体中文与英文两种语言，形如“已重载 N 条配方，耗时 T 毫秒”。

对于 recipes 与 tags 目标，本模组会下发同步数据包，触发 JEI、REI、EMI 等配方查看器自动刷新配方显示。

## 数据包注册表热重载（实验性）

/reloadonly registry <注册表> [confirm] 用于就地热重载单个数据包注册表。该指令更新现有注册表对象及其 Holder 引用，而非替换整个注册表，因此在世界加载时已缓存旧引用的对象（例如伤害来源）能够立即读取到新值，且不会改写世界存档文件。此功能存在以下限制：

- 仅支持运行时查询型（即“叶子型”）注册表，例如 damage_type（伤害类型）、enchantment（附魔，自 Minecraft 1.21 起改为数据包注册表）、trim_pattern（盔甲纹饰）、banner_pattern（旗帜图案）等。
- 生成固化型注册表被列入黑名单并拒绝重载，包括 worldgen 命名空间下的生物群系、结构、密度函数、噪声设置等，以及 dimension_type（维度类型）。此类内容在世界生成阶段已固化写入区块，重载注册表无法追溯修改已生成的区块数据。
- 客户端注册表仅在玩家加入世界时同步，因此重载后玩家需要重新连接服务器才能看到变化。指令在未附带 confirm 参数时会先给出警告，需再次附带 confirm 参数方可执行。

**重载某些类型的数据包注册表可能导致游戏崩溃或存档损坏。在对陌生的数据包注册表使用 /reloadonly registry 前，请务必先备份存档，并在存档副本上进行测试，使用风险自负。**

## 兼容性

以下关于 KubeJS 与 JEI 的说明仅适用于 recipes（配方）目标。recipes 是唯一经过 KubeJS 兼容处理的目标，advancements、tags、loot、functions 均采用原版逻辑。

- 原版及纯 JSON 配方（原版配方、模组内置配方、世界数据包配方）可被完整重载，包括 Create（机械动力）、Mekanism（通用机械）等模组注册的自定义配方类型；配方类型在游戏启动阶段注册，不受重载影响。
- 安装 KubeJS 后，指令会先复现 KubeJS 的脚本重载步骤再触发配方应用，因此脚本配方能够自动生效。Forge 平台使用 KubeJS 6 代，NeoForge 平台使用 KubeJS 7 代。当兼容层发生异常时，会自动回落至原版策略并给出提示。
- /reloadrecipes 会同时下发标签与配方两个同步数据包，使 JEI 等配方查看器自动刷新。JEI 需要在同一周期内同时接收到标签更新事件与配方更新事件才会刷新，因此额外下发的标签数据包用于触发刷新，其标签内容保持不变，且不产生磁盘读写。

## 使用限制

- /reloadonly tags <注册表> 仅重新绑定该注册表的标签，已加载配方中基于标签的材料仍使用旧缓存。修改物品、方块或流体标签后，需接着执行 /reloadrecipes 才能使配方跟随新标签生效。
- /reloadonly functions 仅重建函数库，不连带重载 #function 函数标签；如同时修改了函数标签，需另行重载。
- KubeJS 兼容仅覆盖 recipes 目标，其余目标的 KubeJS 脚本修改在本版本中不会通过本模组生效。
- 本模组不覆盖通过 AddReloadListenerEvent 注册独立监听器的模组，例如 CraftTweaker；此类模组的运行时修改需使用其自带的重载方式或完整的 /reload 指令。
- 在 NeoForge 平台与 KubeJS 7 代环境下，运行时新建的全新命名空间可能无法被读取，这是由 KubeJS 7 代文件资源包的命名空间索引在游戏启动时固定所致；修改已有命名空间下的内容不受此限制。
